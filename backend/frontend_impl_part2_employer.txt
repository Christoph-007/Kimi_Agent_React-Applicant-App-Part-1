### SCR-17 ‚Äî Employer Dashboard (`/employer/dashboard`)

#### A. Design Elements
| Property | Value |
|----------|-------|
| Layout | Sidebar + main; stat cards row (6 cards) + 3 widget sections below |
| Colors | Stat card accents: indigo (jobs), green (apps), amber (pending), blue (shortlist) |
| Responsive | Cards 3√ó2 on desktop ‚Üí 2√ó3 tablet ‚Üí 1-col mobile; sidebar collapses |

#### B. UI Components
| Component | Type | States |
|-----------|------|--------|
| `Sidebar` (Employer) | Reusable | Expanded, collapsed |
| `StatCard` (√ó6) | Reusable | Skeleton, loaded, zero |
| `PendingApprovalBanner` | Screen-specific | Shown if `isApproved = false` |
| `ApplicationCard` (√ó5) | Reusable | Default, hover |
| `JobRequestCard` (√ó3) | Reusable | Default |
| `ShiftCard` (√ó3) | Reusable | Default |
| `QuickActionButton` | Screen-specific | Default, hover |

#### C. Functional Elements
- **API Calls (parallel on mount):**
  - `GET /api/jobs/employer/my-jobs?limit=5`
  - ‚öÝÔ∏è **FIND-01 Workaround:** No `GET /api/applications/employer/recent` endpoint exists in backend. Show recent applications by iterating the first 2-3 job IDs returned above, then calling `GET /api/applications/job/:jobId?limit=2` per job in parallel. Alternatively, display per-job application counts only and link to SCR-21.
  - `GET /api/shifts/employer/my-shifts?limit=3`
  - `GET /api/job-requests/employer/sent?limit=3`
  - `GET /api/notifications/unread-count`
  - `GET /api/shortlist?limit=1` (count only)
- **Approval check:** If `isApproved === false`, show `PendingApprovalBanner`; hide action buttons for job creation
- **Loading:** Independent skeleton per widget
- **Empty States:** Each widget shows contextual empty message + CTA

#### D. Navigation & Routing
- **Route:** `/employer/dashboard`
- **Guard:** `ProtectedRoute role="employer"` (approved + unapproved allowed)
- **Redirect:** Unauthenticated ‚Üí `/login`

#### E. Implementation Notes
- **Folder:** `src/pages/employer/DashboardPage.jsx`
- **Performance:** `Promise.all` for parallel fetches; cache `isApproved` in `AuthContext`

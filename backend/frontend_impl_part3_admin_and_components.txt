### SCR-31 — Admin Dashboard (`/admin/dashboard`)

#### A. Design Elements
| Property | Value |
|----------|-------|
| Layout | Sidebar + main; stat cards row (7 cards) + pending approvals list + recent jobs table + chart |
| Colors | Stat cards: indigo/green/amber/red accent borders; chart: Recharts with brand palette |
| Responsive | Cards 4-col → 3-col → 2-col → 1-col; chart full-width |

#### B. UI Components
| Component | Type | States |
|-----------|------|--------|
| `Sidebar` (Admin) | Reusable | Expanded, collapsed |
| `StatCard` (×7) | Reusable | Skeleton, loaded |
| `PendingEmployerList` | Screen-specific | Loading, populated, empty |
| `RecentJobsTable` | Screen-specific | Loading, populated |
| `ActivityLineChart` (Recharts) | Screen-specific | Loading skeleton, rendered |
| `ApproveRejectButtons` | Reusable | Default, loading |

#### C. Functional Elements
- **API (single call on mount — response contains everything needed):**
  - `GET /api/admin/dashboard/stats` ⚠️ (not `/api/admin/stats`) — returns embedded `recentJobs` and `recentApplications`; no separate jobs-list call needed
  - `GET /api/admin/employers?isApproved=false&limit=5` — for pending approvals widget
- **Approve Employer:** Inline `PUT /api/admin/employers/:id/approve` → remove from pending list
- **Reject Employer:** Inline `PUT /api/admin/employers/:id/reject { reason }` → remove from list
- **Chart:** Renders application counts over time from stats data

#### D. Navigation & Routing
- **Route:** `/admin/dashboard`
- **Guard:** `ProtectedRoute role="admin"`
- **Redirect:** Unauthenticated → `/login`; wrong role → `/`

#### E. Implementation Notes
- **Folder:** `src/pages/admin/DashboardPage.jsx`
- **Libraries:** `recharts` for activity chart
- **Accessibility:** Charts include `title` and `desc` SVG elements for screen readers

- **Loading:** Table skeleton rows

#### D. Navigation & Routing
- **Route:** `/admin/employers`
- **Guard:** `ProtectedRoute role="admin"`

#### E. Implementation Notes
- **Folder:** `src/pages/admin/ManageEmployersPage.jsx`
- **⚠️ Flag:** Critical screen — approval/blocking directly affects employer access to the platform

### SCR-34 — Moderate Jobs (`/admin/jobs`)

#### A. Design Elements
| Property | Value |
|----------|-------|
| Layout | Filter bar + data table |
| Colors | Status badges same as public/employer views |
| Responsive | Table scrollable on mobile |

#### B. UI Components
| Component | States |
|-----------|--------|
| `DataTable` | Loading, populated, empty |
| `StatusBadge` | open, closed, filled |
| `SearchBar` | Default, active |
| `SelectFilter` (status) | Default, selected |
| `ConfirmDialog` (delete) | Closed, open |
| `Pagination` | Standard |

#### C. Functional Elements
- **API:**
  - `GET /api/jobs?page=&status=&search=` ⚠️ Use the **public jobs endpoint** — no `GET /api/admin/jobs` list route exists in backend
  - `DELETE /api/admin/jobs/:id` — the **only** admin job action available
  - ❌ **No `PUT /api/admin/jobs/:id/close` endpoint exists** — do NOT show a "Close Job" button; admin can only delete
- **View Details:** Opens `/jobs/:id` in new tab (public view)
- **Delete:** Requires confirm dialog with "This cannot be undone" warning
- **Loading:** Skeleton table rows

#### D. Navigation & Routing
- **Route:** `/admin/jobs`
- **Guard:** `ProtectedRoute role="admin"`
- **Exit:** `/jobs/:id` (new tab, public view)

#### E. Implementation Notes
- **Folder:** `src/pages/admin/ModerateJobsPage.jsx`

### SCR-35 — Platform Analytics (`/admin/analytics`)

#### A. Design Elements
| Property | Value |
|----------|-------|
| Layout | Date-range controls at top + chart grid (2-col on desktop, 1-col mobile) |
| Colors | Recharts palette: indigo, green, amber, red, purple |
| Responsive | Charts full-width on mobile |

#### B. UI Components
| Component | Type | States |
|-----------|------|--------|
| `DateRangePicker` | Reusable | Default, selecting |
| `PeriodToggle` (daily/weekly/monthly) | Screen-specific | Active tab highlighted |
| `LineChart` (Recharts) | Screen-specific | Loading skeleton, rendered |
| `PieChart` (Recharts) | Screen-specific | Loading, rendered |
| `BarChart` (Recharts) × 2 | Screen-specific | Loading, rendered |
| `DonutChart` (Recharts) | Screen-specific | Loading, rendered |
| `ExportButton` | Reusable | Default, downloading |

#### C. Functional Elements
- **Charts:**

| Chart | Type | Data |
|-------|------|------|
| Applications over time | Line | Daily/weekly count |
| Jobs by type | Pie | full-time/part-time/shift/contract |
| Employer signups | Bar | Monthly |
| Applicant signups | Bar | Monthly |
| Shift completion rate | Donut | Completed vs cancelled |

- **API:**
  - `GET /api/admin/dashboard/stats` ⚠️ (not `/api/admin/stats`) — use this corrected URL
  - `GET /api/admin/analytics` (if available; otherwise derive all chart data from dashboard/stats response)
- **Date range filter:** Re-fetches all chart data on change
- **Export:** CSV/PDF download of current analytics data
- **Loading:** Skeleton chart placeholders

#### D. Navigation & Routing
- **Route:** `/admin/analytics`
- **Guard:** `ProtectedRoute role="admin"`

#### E. Implementation Notes
- **Folder:** `src/pages/admin/AnalyticsPage.jsx`
- **Libraries:** `recharts`; `jspdf` + `papaparse` for export
- **Performance:** Memoize chart data transforms with `useMemo`
- **Accessibility:** Charts provide data tables as accessible alternatives (`<details>` / `<summary>`)

## ⚠️ Screens Requiring Special Attention

| Screen | Reason |
|--------|--------|
| SCR-02 Login | Auth sensitive; JWT storage; role-based redirect |
| SCR-03 Applicant Signup | File upload + multi-step API (signup → resume upload) |
| SCR-04 Employer Signup | Approval flow; unapproved state persists across sessions |
| SCR-10 My Shifts | Browser geolocation required for check-in/out |
| SCR-17 Employer Dashboard | ⚠️ No single "recent applications" endpoint — requires multi-job workaround |
| SCR-21 Job Applications | Complex state: accept/reject(`rejectionReason`)/shortlist/job-request all on same screen |
| SCR-22 Browse Applicants | Saved filters must persist in DB, not just local state |
| SCR-31 Admin Dashboard | ⚠️ Use `/api/admin/dashboard/stats` (not `/api/admin/stats`); recentJobs embedded in response |
| SCR-32 Manage Employers | Critical — approve/block actions have immediate business impact |
| SCR-33 Manage Applicants | ⚠️ Admin can only **deactivate** — no activate endpoint; do NOT implement "Activate" button |
| SCR-34 Moderate Jobs | ⚠️ No admin jobs list endpoint; use public `GET /api/jobs`; no close-job action |
| SCR-35 Analytics | ⚠️ Use `/api/admin/dashboard/stats`; chart performance: memoize transforms |
